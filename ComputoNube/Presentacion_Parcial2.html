¿Qué es la Virtualización?

Unidad III · 3.1 — Fundamentos y tipos de hipervisores
🖥️

📖 Definición

La virtualización es la tecnología que permite crear representaciones virtuales de recursos físicos (servidores, almacenamiento, redes), permitiendo que un solo hardware físico ejecute múltiples sistemas operativos o aplicaciones de forma aislada y simultánea.

Metáfora: como dividir una casa en departamentos: el mismo edificio físico alberga múltiples hogares independientes, cada uno con sus propios servicios.

Hipervisor Tipo 1 vs Tipo 2

Tipo 1 — Bare Metal (sobre el hardware)

Corre directamente sobre el hardware físico sin OS anfitrión. Mayor rendimiento y seguridad. Usado en producción empresarial.

Ejemplos: VMware ESXi, Microsoft Hyper-V, Proxmox VE, KVM, Xen

Metáfora: el arquitecto que diseña directamente sobre el terreno.

Tipo 2 — Hosted (sobre un OS anfitrión)

Se instala sobre un sistema operativo existente (Windows, Linux, macOS). Mayor compatibilidad pero menor rendimiento. Ideal para desarrollo y aprendizaje.

Ejemplos: VirtualBox, VMware Workstation Player, Parallels

Metáfora: construir una maqueta dentro de una habitación ya existente.

Beneficios de la virtualización

💰
Reducción de costos: un servidor físico puede ejecutar 10-20 VMs, reduciendo drásticamente el hardware necesario.
⚡
Mayor eficiencia: los servidores tradicionales usan ~15% de su capacidad; con virtualización suben al 70-80%.
🔄
Alta disponibilidad: si un servidor físico falla, las VMs migran automáticamente a otro (Live Migration).
🔐
Aislamiento: cada VM es independiente; un virus en una VM no afecta a las demás.
📸
Snapshots: guardar el estado exacto de una VM en un momento dado. Respaldo instantáneo antes de cambios críticos.
🌱
Sustentabilidad: menos hardware físico = menor consumo eléctrico y menor huella de carbono (Green IT).

Tipos de Virtualización

Unidad III · 3.1 — Infraestructura completa virtualizada
🌐

Haz clic en cada tipo para ver sus detalles

🖥️ Virtualización de Servidor — el más común
Múltiples VMs corriendo en un solo servidor físico. Cada VM tiene su propio OS, CPU virtual, RAM y almacenamiento asignado. Ejemplo: un servidor físico Dell con 64GB RAM corre 8 VMs de 8GB cada una para diferentes departamentos de TechMX. Herramientas: VMware ESXi, Proxmox, Hyper-V, KVM.
💾 Virtualización de Almacenamiento — abstrae el hardware de discos
Abstrae múltiples discos físicos (SATA, SAS, SSD) en un pool unificado gestionable. Permite crear volúmenes lógicos independientes del hardware. Ejemplo: 10 discos de 1TB se presentan al OS como un solo disco virtual de 10TB con redundancia. Tecnologías: RAID virtual, SAN, NAS, VMware VSAN, Ceph.
🌐 Virtualización de Red — SDN (Software Defined Networking)
Crea redes lógicas independientes sobre la misma infraestructura física de red. Permite segmentar tráfico, crear VLANs, y gestionar redes desde software. Ejemplo: TechMX puede tener red de producción, red de desarrollo y red de gestión completamente aisladas sobre los mismos switches físicos. Herramientas: VMware NSX, Open vSwitch, VLANs, Azure VNet.
🖥️ Virtualización de Escritorio (VDI) — Virtual Desktop Infrastructure
Los escritorios de los empleados corren como VMs en el servidor centralizado; el usuario solo necesita un thin client. Ejemplo: Los 50 empleados de TechMX acceden a su Windows 11 virtualizado desde cualquier dispositivo. Ventaja: gestión centralizada, menor costo de endpoints. Herramientas: VMware Horizon, Citrix, Azure Virtual Desktop.
📦 Virtualización de Aplicaciones — aislamiento sin instalación
La aplicación se empaqueta con sus dependencias y corre en un entorno aislado sin instalación tradicional. Evita conflictos de versiones. Ejemplo: TechMX corre Office 2019 y Office 365 simultáneamente en el mismo equipo sin conflictos. Herramientas: Microsoft App-V, VMware ThinApp, Citrix XenApp, Docker (contenedores).
🔄 Sistemas Operativos Distribuidos — múltiples nodos como uno solo
Múltiples computadoras trabajando como un único sistema coherente. El OS gestiona recursos distribuidos de forma transparente para el usuario. Ejemplo: El clúster de servidores de TechMX en la nube aparece como una sola máquina de gran capacidad. Tecnologías: Kubernetes, Hadoop, Apache Spark, Proxmox Cluster.
💡 Metáfora del edificio de apartamentos

Virtualización de servidor = dividir el edificio en apartamentos. Almacenamiento = el estacionamiento compartido donde cada inquilino tiene su cajón asignado. Red = los intercomunicadores y pasillos separados por piso. Escritorio = el portero que entrega el mismo "apartamento virtual" a cualquier inquilino desde cualquier sucursal. Aplicaciones = los electrodomésticos portátiles que el inquilino lleva a cualquier apartamento.

Proxmox VE — Virtualización Empresarial

Unidad III · 3.1.3 — Plataforma open source Tipo 1
🏗️

🔍 ¿Qué es Proxmox VE?

Proxmox Virtual Environment es una plataforma de virtualización empresarial open source basada en Debian Linux. Combina virtualización con KVM (VMs completas) y contenedores LXC en una sola interfaz web. Ideal para servidores de producción y laboratorios avanzados.

  • 100% gratuito (soporte comercial opcional)
  • Interfaz web potente para gestión completa
  • Clustering nativo: múltiples nodos como una sola plataforma
  • Alta disponibilidad (HA) automática
  • Backup integrado (Proxmox Backup Server)

KVM vs Contenedores LXC en Proxmox

Alta Disponibilidad (HA) en Proxmox

¿Qué es un clúster HA?

Múltiples nodos Proxmox trabajando juntos. Si un nodo falla, las VMs migran automáticamente a otro nodo activo sin intervención manual.

Metáfora: como un equipo de cirujanos donde si el cirujano principal se desmaya, otro toma el bisturí automáticamente sin que el paciente note la diferencia.

Requisitos mínimos para HA

  • Mínimo 3 nodos (para quórum de votación)
  • Almacenamiento compartido (Ceph, NFS, iSCSI)
  • Red de gestión dedicada (Corosync)
  • Fencing: mecanismo para aislar nodo fallido

Laboratorio 2: Proxmox en VirtualBox (nested virtualization)

  • Crear VM en VirtualBox con 4GB RAM, 40GB disco, habilitar VT-x/AMD-V
  • Instalar Proxmox VE desde ISO (proxmox.com/downloads)
  • Acceder a la interfaz web: https://IP:8006
  • Crear VM Ubuntu dentro de Proxmox (virtualización anidada)
💡 Proxmox para TechMX

Si TechMX decide implementar una nube privada antes de migrar completamente a Azure, Proxmox es la opción ideal: costo cero de licencias, interfaz web fácil de usar, y escalable de 1 servidor a un clúster de 10 nodos sin cambio de plataforma. Muchas empresas mexicanas usan Proxmox como nube privada de transición.

🔵 Azure Virtual Machines

El servicio IaaS más popular de Azure. Permite crear VMs Windows o Linux en minutos, elegir el tamaño (CPU/RAM), configurar redes virtuales y almacenamiento, todo desde el portal web o mediante CLI/API.

Laboratorio 3: Primera VM en Azure for Students

1
Activar Azure for Students: azure.microsoft.com/es-mx/free/students → usar correo institucional IPN. Obtienes $100 USD de crédito.

Crear VM: Máquinas Virtuales → Crear → Ubuntu Server 22.04, tamaño B1s (1 vCPU, 1GB RAM — costo mínimo), autenticación SSH.

5
Conectar y documentar: SSH a la VM pública, instalar Apache, capturar pantalla de la web funcionando. ⚠️ Apagar la VM cuando no se use para conservar créditos.

Escenarios de Gestión Virtualizada

Unidad III · 3.2 — Monitoreo · Clonación · Almacenamiento
⚙️

📊 Monitoreo de Procesos

¿Qué monitorear?

  • CPU: % de uso por VM, tiempo de espera (steal time)
  • RAM: uso, swap, ballon driver
  • Disco: IOPS, latencia de lectura/escritura
  • Red: ancho de banda, paquetes perdidos
  • Estado del hipervisor: temperatura, energía

Herramientas de monitoreo

  • Proxmox: panel web integrado, Grafana
  • Azure: Azure Monitor, Log Analytics
  • VirtualBox: estadísticas de VM integradas
  • Linux: htop, iotop, nethogs

Metáfora: el tablero de instrumentos del avión — muestra en tiempo real si todos los motores funcionan correctamente.

📋 Clonación de Servicios

Tipos de clonación

  • Clon completo: copia independiente total de la VM. Ocupa el mismo espacio en disco.
  • Clon enlazado: usa el disco base de origen; solo guarda diferencias. Ahorra espacio pero más lento.
  • Template/Plantilla: VM maestra sellada para crear múltiples clones rápidamente.

Caso de uso TechMX

Crear una plantilla "Ubuntu-Web-TechMX" con Apache y las dependencias instaladas. Clonarla en 5 minutos para cada nuevo servidor web necesario, en lugar de instalar desde cero (4 horas cada uno).

Metáfora: como la impresora 3D que puede producir el mismo objeto n veces a partir de un solo diseño.

💾 Almacenamiento Virtual

Tipos de almacenamiento en VMs

  • VDI/VMDK/VHD: formatos de disco virtual (VirtualBox/VMware/Hyper-V)
  • Thin provisioning: el disco crece según se usa (ahorra espacio)
  • Thick provisioning: reserva todo el espacio al crear (mejor rendimiento)
  • NFS/iSCSI: almacenamiento compartido entre VMs en red

Snapshots (Instantáneas)

Guardan el estado completo de la VM (RAM + disco + configuración) en un momento dado. Permiten revertir cambios en segundos.

Metáfora: el punto de guardado en un videojuego antes de entrar al boss final.

💡 Los 3 escenarios aplicados a TechMX

Monitoreo: dashboard en Azure Monitor que alerta cuando el CPU del servidor de BD supera 80%. Clonación: plantilla "WebServer-TechMX" para desplegar nuevos servidores en 5 minutos cuando haya picos de demanda. Almacenamiento: discos Azure Managed con geo-redundancia para que los datos de TechMX sobrevivan un fallo del data center completo.

Evaluación de Entornos Virtualizados

Unidad III · 3.3 — 4 dimensiones de evaluación
📊

Las 4 dimensiones de evaluación (criterios 3.3)

⚡ Desempeño

¿Qué tan rápido y eficiente es el entorno?

  • Latencia de I/O de disco (ms)
  • Overhead del hipervisor (% CPU extra)
  • Throughput de red (Mbps)
  • Tiempo de arranque de la VM
  • Rendimiento bajo carga (benchmark)

🔧 Funcionalidad

¿Qué características y capacidades ofrece?

  • Tipos de virtualización soportados
  • Herramientas de gestión disponibles
  • Soporte de snapshots y clonación
  • Integración con herramientas externas
  • Consola de gestión (web/GUI/CLI)

 

🔄 Adaptabilidad

¿Qué tan flexible y escalable es?

  • Escalabilidad horizontal (agregar nodos)
  • Compatibilidad con múltiples OS
  • Integración con cloud (híbrido)
  • Portabilidad de VMs entre plataformas
  • Soporte de automatización (APIs, Ansible)

🔐 Seguridad

¿Qué tan seguro es el entorno?

  • Aislamiento entre VMs
  • Control de acceso basado en roles
  • Cifrado de datos en reposo y tránsito
  • Auditoría y logs de actividad
  • Certificaciones de seguridad del proveedor

Quiz — Evaluación aplicada

TechMX necesita que si un servidor falla, las VMs migren automáticamente a otro sin interrumpir a los usuarios. ¿Qué dimensión de evaluación es la más crítica para esta decisión?

✅ ¡Correcto! La Funcionalidad es la más crítica aquí: necesitas que la plataforma tenga la función de Alta Disponibilidad (HA) y Live Migration. Proxmox y VMware ESXi la tienen; VirtualBox no. Esta función es lo que habilita la migración automática al fallar un nodo.
❌ Revisa. La migración automática ante fallos es una función específica del hipervisor (Alta Disponibilidad). Esto corresponde a la dimensión de Funcionalidad. El desempeño, adaptabilidad y seguridad también importan, pero la HA es una función que la plataforma tiene o no tiene.

Contenedores vs Máquinas Virtuales

Unidad III — Docker, Kubernetes y su relación con la nube
📦

🖥️ Máquina Virtual (VM)

Emula hardware completo: tiene su propio kernel, OS completo, librerías y aplicación. Aislamiento total. Mayor overhead (tamaño: 1-10GB+).

Metáfora: una casa independiente con su propio sistema de agua, luz y gas.

  • ✅ Aislamiento completo (Windows + Linux en el mismo host)
  • ✅ Más seguro para workloads críticos
  • ❌ Arranque lento (minutos)
  • ❌ Mayor consumo de recursos (RAM, disco)

📦 Contenedor (Docker/LXC)

Comparte el kernel del OS anfitrión pero aísla la aplicación con sus dependencias. Ultra-ligero (tamaño: 10-500MB). Arranque en segundos.

Metáfora: departamentos en un edificio: comparten la estructura pero cada uno es independiente.

  • ✅ Ultra-rápido y ligero
  • ✅ Portabilidad total: "corre igual en cualquier parte"
  • ❌ Solo Linux (o WSL2 en Windows)
  • ❌ Menor aislamiento que VMs

Kubernetes — Orquestación de contenedores

¿Qué es Kubernetes (K8s)?

Sistema open source de Google para automatizar el despliegue, escalado y gestión de aplicaciones en contenedores. Gestiona clústeres de contenedores Docker/Containerd.

Casos de uso

  • Escalar automáticamente una app web cuando hay picos de tráfico
  • Actualizar una aplicación sin downtime (rolling updates)
  • Recuperar automáticamente contenedores que fallan

Kubernetes en Azure

AKS (Azure Kubernetes Service): Kubernetes gestionado en Azure. TechMX puede desplegar sus microservicios en contenedores y que Azure gestione automáticamente la infraestructura subyacente.

💡 VM vs Contenedor en TechMX

Usar VMs para: el servidor de BD Oracle (necesita Windows Server específico) y el servidor de archivos. Usar contenedores para: la aplicación web del ERP (Docker en Azure AKS) donde necesitan escalar rápido y hacer deployments frecuentes. Los contenedores son el futuro del cloud moderno.
