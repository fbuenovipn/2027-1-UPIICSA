Sesión 21 — NLP y Transformers: El Lenguaje de la Industria

🎭 METÁFORA

Antes de los Transformers, los modelos de lenguaje leían texto como alguien que lee una carta cubriéndola con la mano: solo veían una palabra a la vez. El mecanismo de atención en los Transformers es como leer toda la carta simultáneamente y poder mirar cualquier parte en cualquier momento. Eso cambió todo.

2017
"Attention Is All You Need" — el paper que cambió la NLP (Google Brain)
BERT
Primer modelo preentrenado masivo de comprensión bidireccional (Google, 2018)
GPT-4
Estado del arte en generación de texto, razonamiento y código

TF-IDF — La Base

Term Frequency × Inverse Document Frequency. Representa texto como vector numérico ponderado. Simple, interpretable, funciona bien para clasificación de documentos cortos y búsqueda.

Word Embeddings

Word2Vec, GloVe, FastText: representaciones densas que capturan significado semántico. "rey - hombre + mujer ≈ reina". La base conceptual de los LLMs modernos.

Transformers y BERT

Atención multi-cabeza: cada token atiende a todos los demás simultáneamente. BERT aprende representaciones bidireccionales preentrenando en enorme corpus. Fine-tuning con pocos datos propios.

Clasificación de Texto con Hugging Face Transformers

Pipeline de Inferencia (Zero-Shot)

from transformers import pipeline # Clasificación sin entrenamiento adicional clasificador = pipeline( "zero-shot-classification", model="facebook/bart-large-mnli") texto = """El compresor C-101 presenta vibración excesiva a 50Hz y temperatura de 95°C en los rodamientos. ❓ QUIZ — Click para ver respuesta

Tienes 10,000 reportes históricos de fallas en texto libre (en español) etiquetados con tipo de falla (4 categorías). ¿Usarías TF-IDF + Random Forest o fine-tuning de BERT? ¿Por qué?

Dependería: con 10,000 ejemplos y texto relativamente corto y técnico, ambas opciones pueden funcionar bien. TF-IDF + RF: más rápido de entrenar, interpretable, sin GPU requerida. Fine-tuning BERT: captura matices semánticos, maneja mejor abreviaturas y jerga técnica no vista. Recomendación: empezar con TF-IDF + RF como baseline (fast to implement), luego comparar con BERT fine-tuned. Si la mejora justifica el costo computacional (+5% F1), usar BERT en producción.

Sesión 22 — Deep Learning: CNN, PyTorch y Transfer Learning

🎭 METÁFORA

Una red neuronal profunda aprende como un niño que aprende a reconocer objetos: primero detecta bordes, luego formas, luego partes, finalmente objetos completos. Las capas convolucionales (CNN) son detectores de patrones jerárquicos: de píxeles a texturas a formas a objetos. Transfer Learning es como contratar a alguien que ya sabe ver y solo entrenarle para tu problema específico.

Redes Neuronales Profundas

Capas de neuronas artificiales con funciones de activación no lineales. Aprenden representaciones jerárquicas de los datos. Requieren: muchos datos, GPU, regularización cuidadosa.

CNN — Redes Convolucionales

Especializadas en datos con estructura espacial: imágenes, señales de sensores 1D. Capas: Conv2D (detectar patrones) → MaxPool (reducir) → Dense (clasificar). Estado del arte en visión computacional.

Transfer Learning

Usar un modelo preentrenado en millones de imágenes (ResNet, EfficientNet) y re-entrenar solo las últimas capas con tus pocos datos. Funciona con 100-1,000 imágenes. El "hack" más poderoso del deep learning.

Inspección Visual de Calidad con Transfer Learning

Fine-tuning ResNet50 con PyTorch

import torch import torch.nn as nn import torchvision.transforms as transforms from torchvision import models, datasets from torch.utils.data import DataLoader # Transformaciones de imagen transform = transforms.Compose([ transforms.Resize((224, 224)), transforms.RandomHorizontalFlip(), transforms.ColorJitter(brightness=0.2), transforms.ToTensor(), transforms.Normalize([0.485,0.456,0.406], [0.229,0.224,0.225])]) # Dataset de imágenes de piezas (defecto/ok) train_data = datasets.ImageFolder( 'data/piezas/train/', transform=transform) test_data = datasets.ImageFolder( 'data/piezas/test/', transform=transform) train_loader = DataLoader(train_data, batch_size=32, shuffle=True, num_workers=4) # Cargar ResNet50 preentrenado modelo = models.resnet50(pretrained=True) # Congelar todas las capas excepto las últimas for param in modelo.parameters(): param.requires_grad = False # Reemplazar última capa para 2 clases num_ftrs = modelo.fc.in_features modelo.fc = nn.Sequential( nn.Dropout(0.4), nn.Linear(num_ftrs, 2)) # ok / defecto device = torch.device("cuda" if torch.cuda.is_available() else "cpu") modelo = modelo.to(device)

Entrenamiento y Evaluación

criterion = nn.CrossEntropyLoss() optimizer = torch.optim.Adam( modelo.fc.parameters(), lr=0.001) scheduler = torch.optim.lr_scheduler.StepLR( optimizer, step_size=7, gamma=0.1) # Bucle de entrenamiento for epoch in range(15): modelo.train() running_loss, correct = 0.0, 0 for inputs, labels in train_loader: inputs, labels = inputs.to(device), \ labels.to(device) optimizer.zero_grad() outputs = modelo(inputs) loss = criterion(outputs, labels) loss.backward() optimizer.step() running_loss += loss.item() _, preds = torch.max(outputs, 1) correct += (preds == labels).sum().item() scheduler.step() acc = correct / len(train_data) print(f"Época {epoch+1}: Loss={running_loss:.3f}, Acc={acc:.1%}") # Guardar modelo torch.save(modelo.state_dict(), 'resnet_calidad_v1.pth')

Sesión 23 — Reinforcement Learning: El Agente que Aprende Jugando

🎭 METÁFORA

Reinforcement Learning es como entrenar a un perro con premios y castigos, pero el perro es un algoritmo y puede intentar millones de veces por hora. El agente no aprende de ejemplos etiquetados, sino de la consecuencia de sus acciones en un entorno. Gana recompensas por buenas decisiones, penalidades por malas. Con suficiente práctica, aprende estrategias que ningún humano había imaginado.

El Paradigma RL

Agente observa el estado del entorno, toma una acción, recibe una recompensa y pasa al siguiente estado. El objetivo: maximizar la recompensa acumulada a largo plazo.

Q-Learning

Tabla Q(s,a) que estima el valor esperado de tomar acción a en estado s. Se actualiza con la ecuación de Bellman. Simple, funciona para espacios pequeños de estados y acciones.

Deep Q-Network (DQN)

Reemplaza la tabla Q con una red neuronal. Permite estados continuos y de alta dimensión (imágenes, sensores). Inventado por DeepMind para jugar Atari a nivel humano (2013).

RL en Python: Gymnasium + Stable Baselines 3

Entorno Gymnasium y Primeros Pasos

import gymnasium as gym import numpy as np # CartPole — equilibrar un palo sobre un carro env = gym.make("CartPole-v1", render_mode="human") # Anatomía del entorno print(f"Espacio de estados: {env.observation_space}") # Box(4,) — posición, velocidad, ángulo, vel. angular print(f"Espacio de acciones: {env.action_space}") # Discrete(2) — empujar izq o der # Episodio con política aleatoria (baseline) obs, info = env.reset(seed=42) total_reward = 0 for step in range(500): action = env.action_space.sample() # aleatorio obs, reward, terminated, truncated, info = env.step(action) total_reward += reward if terminated or truncated: print(f"Episodio terminó en paso {step+1}") print(f"Recompensa total: {total_reward}") obs, info = env.reset() break env.close()

Sesión 24 — Taller del Proyecto Integrador de IA

🎭 METÁFORA

El proyecto integrador es el examen de conducir del seminario. Puedes haber memorizado el reglamento de tránsito (la teoría), pero demuestras competencia real cuando manejas en la ciudad (un problema real de tu industria) sin instructor. Es donde la teoría se convierte en habilidad profesional.

Requisitos del Proyecto

  • 1

    Problema de negocio real

    Formulación clara del problema, KPI objetivo, impacto económico estimado de la solución

  • 2

    Pipeline de datos completo

    Fuentes, limpieza, EDA documentado, feature engineering justificado

  • 3

    Modelo(s) ML/DL

    Al menos 2 algoritmos comparados con métricas estándar y justificación de la elección final

  • 4

    Visualización y comunicación

    Dashboard interactivo (Plotly, Power BI o Streamlit) con hallazgos clave para tomadores de decisión

  • 5

    Reproducibilidad

    Repositorio Git con requirements.txt o environment.yml. Notebook documentado en Google Colab.

Ejemplos de Proyectos por Sector

  • Manufactura: predicción de fallas en línea de producción (mantenimiento predictivo)
  • Logística: optimización de rutas de última milla con datos históricos + algoritmo genético
  • Retail/FMCG: predicción de demanda por SKU/punto de venta + análisis de canibalización
  • Energía: detección de anomalías en consumo eléctrico (posible fraude o equipo defectuoso)
  • Agro-industria: predicción de rendimiento por parcela usando imágenes satelitales + variables de suelo
  • Salud/Farma: clasificación de incidentes de calidad en reportes de texto (NLP)
  • Banca: scoring de crédito con interpretabilidad (SHAP) para cumplimiento regulatorio
  • Construcción: estimación de costos de proyecto con Random Forest + análisis de desviaciones históricas

Estructura del Notebook Final

📋 SECCIÓN 1 — EXECUTIVE SUMMARY: Problema, solución propuesta, impacto esperado (1 página para directivos)
📊 SECCIÓN 2 — EDA: Descripción del dataset, estadísticas descriptivas, visualizaciones clave, hipótesis de negocio
🔧 SECCIÓN 3 — PREPROCESAMIENTO: Limpieza, imputación, encoding, feature engineering con justificación de cada decisión
🤖 SECCIÓN 4 — MODELADO: Baseline + mínimo 2 modelos comparados con cross-validation y métricas apropiadas
🔍 SECCIÓN 5 — INTERPRETABILIDAD: SHAP values, feature importance, casos de error (análisis de falsos positivos/negativos)
🚀 SECCIÓN 6 — DEPLOYMENT: API FastAPI o app Streamlit funcionando. Instrucciones de uso.
💡 DATO CLAVE — Lo que diferencia un buen proyecto

Los mejores proyectos no son los con el modelo más complejo, sino los que conectan cada decisión técnica con el impacto de negocio. "Elegí recall como métrica principal porque un falso negativo (falla no detectada) cuesta $50,000 en parada de planta, mientras que un falso positivo (mantenimiento innecesario) solo cuesta $500." Eso es pensar como ingeniero de datos industrial.

Sesión 25 — Defensa Final: Cómo Presentar tu Proyecto de IA

🎭 METÁFORA

Una defensa de proyecto de IA es como vender una solución a un directivo escéptico en 15 minutos. El directivo no entiende de gradiente descendente ni de cross-entropy loss. Lo que sí entiende es: ¿cuánto dinero ahorra?, ¿cómo funciona en términos simples?, ¿qué pasa si falla? Tu trabajo es traducir ciencia en valor empresarial.

Estructura de la Presentación (15 min)

  1. Problema y contexto (2 min)
  2. Datos utilizados (1 min)
  3. Solución técnica simplificada (3 min)
  4. Resultados medibles (3 min)
  5. Demo en vivo (3 min)
  6. Inversión y ROI estimado (2 min)
  7. Preguntas (5 min adicionales)

Cómo Explicar ML sin Tecnicismos

En lugar de "Random Forest con 200 árboles de decisión y profundidad máxima 10": "Un comité de 200 expertos independientes que cada uno analiza parte del historial de fallas y vota; la mayoría decide si hay riesgo. El sistema aprendió de 3 años de datos históricos."

Manejo de Preguntas Difíciles

  • "¿Y si el modelo se equivoca?" → describir el plan de fallback y monitoreo
  • "¿Por qué no Excel?" → mostrar lo que Excel no puede hacer a esa escala
  • "¿Es seguro para producción?" → hablar de validación, pruebas en paralelo y rollout gradual

🔬 CONSEJOS PARA LA DEFENSA

(1) Demo en vivo con datos reales de tu empresa (impacto máximo). (2) Mostrar comparación "antes/después" del proceso manual vs el modelo. (3) Tener preparada una diapositiva de errores: "El modelo se equivoca en estos casos y aquí está cómo los manejamos." La transparencia sobre limitaciones aumenta la confianza. (4) Traer una propuesta de plan de implementación de 90 días.
