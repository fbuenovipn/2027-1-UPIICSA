❓ QUIZ — Click para ver respuesta

Tienes un DataFrame con 100,000 filas de telemetría de sensores. El 15% de valores de temperatura son NaN. ¿Qué harías antes de entrenar un modelo con esos datos?

Primero investiga el patrón de missings: ¿son aleatorios (MAR), completamente aleatorios (MCAR) o dependen de otra variable (MNAR)? Si son MCAR/MAR: imputar con la mediana (robusta ante outliers) o con el valor del sensor más cercano en el tiempo (forward/backward fill para series temporales). Crear una columna binaria "temp_imputed" como indicador. Si son MNAR: el motivo de la falla puede ser información valiosa — considerar modelar la ausencia. Nunca eliminar filas si son 15% de datos (pérdida significativa).

Sesión 8 — SQL y Bases de Datos Relacionales para el Científico de Datos

🎭 METÁFORA

SQL es como el idioma universal de las empresas. No importa si los datos están en Oracle, PostgreSQL, MySQL o SQL Server: la gramática es la misma. El 80% de los datos empresariales viven en bases de datos relacionales. No saber SQL como científico de datos es como ser mecánico y no saber usar la llave de tuercas.

Modelo Relacional

Datos organizados en tablas con filas y columnas. Las relaciones entre tablas se establecen mediante claves primarias (PK) y foráneas (FK). Elimina redundancia y garantiza integridad.

SQLite en Python

Base de datos relacional embebida. Sin servidor, un solo archivo. Ideal para prototipado, análisis local y proyectos de DS sin infraestructura adicional.

SQLAlchemy ORM

Abstracción de Python sobre SQL. Crea y consulta tablas usando clases Python. Portable entre motores (SQLite, PostgreSQL, MySQL) sin cambiar código.

SQL Esencial para Ciencia de Datos

Consultas Fundamentales

-- Selección y filtrado
SELECT maquina_id, AVG(defectos) as prom_defectos,
       MAX(temperatura) as temp_max
FROM produccion
WHERE fecha >= '2024-01-01'
  AND turno IN ('A', 'B')
GROUP BY maquina_id
HAVING AVG(defectos) > 5
ORDER BY prom_defectos DESC
LIMIT 10;

-- JOIN entre tablas
SELECT p.fecha, p.defectos, o.nombre as operador
FROM produccion p
JOIN operadores o ON p.operador_id = o.id
WHERE p.linea = 'L1';

SQL Avanzado para Análisis

-- Window Functions — estadísticas por ventana
SELECT fecha, linea, produccion,
  AVG(produccion) OVER (
    PARTITION BY linea
    ORDER BY fecha
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
  ) AS promedio_7dias
FROM produccion;

-- CTE — consultas legibles y reutilizables
WITH resumen_turno AS (
  SELECT turno, SUM(produccion) as total
  FROM produccion
  GROUP BY turno
)
SELECT turno, total,
  ROUND(total*100.0/SUM(total) OVER(), 1) as pct
FROM resumen_turno;

Sesión 9 — MongoDB, APIs REST y Web Scraping

🎭 METÁFORA

Si SQL es como una biblioteca bien catalogada con estantes ordenados (tablas fijas, esquema rígido), MongoDB es como un almacén flexible donde cada caja puede tener distinto contenido y forma. Perfecto para datos semiestructurados: logs de eventos, datos de sensores IoT con atributos variables, registros de clientes con campos opcionales.

MongoDB — NoSQL Documental

Almacena documentos JSON (BSON internamente). Sin esquema fijo. Escala horizontalmente. Ideal para datos de IoT, logs, perfiles de usuario, catálogos de productos variables.

APIs REST

Interfaz estándar para intercambiar datos entre sistemas. GET (obtener), POST (enviar), PUT (actualizar), DELETE. Autenticación: API key, OAuth 2.0, JWT. Respuesta típica: JSON.

Web Scraping

Extracción automatizada de datos de sitios web. BeautifulSoup para HTML estático, Selenium para sitios dinámicos con JavaScript. Verificar robots.txt y términos de uso.

Kedro: Pipelines de Datos Reproducibles y Productivos

📂 DATA CATALOG — Define fuentes de datos (CSV, SQL, APIs, Parquet) en YAML. El código no "sabe" dónde están los archivos.
⚙️ NODES — Funciones Python puras: input → transformación → output. Sin efectos secundarios, fáciles de testear.
🔗 PIPELINE — Grafo de dependencias entre nodos. Kedro resuelve el orden automáticamente (DAG). Paralelización automática.
📊 KEDRO-VIZ — Visualización interactiva del pipeline. Ver el flujo de datos completo de un vistazo.
🔬 EXPERIMENTOS — Registro de parámetros, métricas y versiones de datos. MLflow integration.
