Sesión 11 — Preprocesamiento Avanzado y Feature Engineering

🎭 METÁFORA

El feature engineering es como preparar los ingredientes antes de cocinar. Con los mejores ingredientes mal preparados (datos crudos, mal escalados, con categorías sin codificar), incluso el mejor chef (modelo ML) producirá un plato mediocre. La calidad de las features determina el techo del modelo, más que el algoritmo elegido.

Imputación de Datos

Simple: media/mediana/moda. KNN Imputer: usa los k vecinos más similares. IterativeImputer: itera entre variables para imputación multivariada. Sklearn lo hace con una línea de código.

Escalado y Normalización

StandardScaler: z-score (media 0, std 1). Necesario para SVM, redes neuronales, PCA. MinMaxScaler: escala a [0,1]. RobustScaler: usa IQR, robusto ante outliers.

Codificación Categórica

One-Hot: dummy variables (turno A/B/C → 3 columnas binarias). Label Encoder: ordinal (A=0, B=1, C=2). Target Encoder: reemplaza categoría por media del target (evita alta dimensionalidad).

Feature Engineering: Crear Variables que el Modelo No Puede Ver

Features Derivadas de Tiempo

df['hora'] = df['timestamp'].dt.hour
df['dia_semana'] = df['timestamp'].dt.dayofweek
df['es_fin_semana'] = df['dia_semana'].isin([5,6]).astype(int)
df['turno'] = pd.cut(df['hora'],
    bins=[0,8,16,24], labels=['noche','mañana','tarde'])

# Lag features — valores anteriores como predictores
df['prod_t-1'] = df['produccion'].shift(1)   # 1 hora atrás
df['prod_t-24'] = df['produccion'].shift(24) # 24 horas atrás

# Rolling statistics — tendencia reciente
df['media_7d'] = df['produccion'].rolling(7).mean()
df['std_7d'] = df['produccion'].rolling(7).std()

Features de Dominio Industrial

# OEE compuesto desde sus componentes
df['oee'] = (df['disponibilidad'] *
             df['rendimiento'] *
             df['calidad'])

# Temperatura × humedad → índice de estrés térmico
df['stress_termico'] = df['temp'] * df['humedad'] / 100

# Interacciones entre variables
df['vel_x_temp'] = df['velocidad'] * df['temperatura']
df['ratio_defectos'] = df['defectos'] / df['produccion'].clip(1)

# Encoding cíclico de tiempo (para ML)
import numpy as np
df['hora_sin'] = np.sin(2*np.pi*df['hora']/24)
df['hora_cos'] = np.cos(2*np.pi*df['hora']/24)
🔬 LABORATORIO — Pipeline de Preprocesamiento

Usar sklearn Pipeline para encadenar: SimpleImputer → RobustScaler → OneHotEncoder (con ColumnTransformer para tratar numéricas y categóricas por separado) → modelo. Ventaja: fit() en train, transform() en test. Sin data leakage garantizado.

❓ QUIZ — Click para ver respuesta

Tienes 150 variables de sensores para predecir fallas. El modelo Random Forest tiene R²=0.97 en train y R²=0.43 en test. ¿Qué está pasando y cómo lo resuelves?

Overfitting severo. El modelo memorizó el train en lugar de aprender patrones generalizables. Soluciones: (1) Reducir complejidad del modelo (menos árboles, menor profundidad máxima), (2) Feature selection para eliminar variables irrelevantes o ruidosas, (3) Regularización, (4) Más datos de entrenamiento, (5) Cross-validation para detectar overfitting antes de reportar métricas, (6) Verificar data leakage — quizás alguna feature "filtra" el target.

Sesión 12 — Aprendizaje No Supervisado: Clustering Industrial

🎭 METÁFORA

El clustering es como organizar un almacén sin etiquetas. No sabes de antemano cuántas categorías hay ni cuáles son, pero el algoritmo encuentra los grupos "naturales" según la similitud de los objetos. Es el ojo que descubre patrones ocultos que ningún humano habría predefinido: segmentos de clientes, modos de falla, patrones de consumo.

K-Means

Asigna cada punto al centroide más cercano. Rápido, escalable. Requiere especificar K. Sensible a outliers y a la escala. Ideal para segmentación de clientes y clasificación de productos.

DBSCAN

Basado en densidad. No requiere K. Detecta outliers como ruido. Encuentra clusters de forma arbitraria. Ideal para detección de anomalías en telemetría de sensores.

Jerárquico

Construye un dendrograma de fusiones. Permite visualizar la jerarquía de clusters. Útil para datasets medianos y cuando el número de clusters no es claro.

K-Means en la Práctica: Segmentación Industrial

Implementación Completa

from sklearn.cluster import KMeans from sklearn.preprocessing import StandardScaler from sklearn.metrics import silhouette_score import matplotlib.pyplot as plt # SIEMPRE escalar antes de clustering scaler = StandardScaler() X_scaled = scaler.fit_transform(X) # Método del codo para elegir K inercias = [] for k in range(2, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) km.fit(X_scaled) inercias.append(km.inertia_) plt.plot(range(2,11), inercias, 'bo-') plt.xlabel('K'); plt.ylabel('Inercia') plt.title('Método del Codo') # Validar con Silhouette Score (0 a 1, mayor = mejor) km_final = KMeans(n_clusters=4, random_state=42, n_init=10) labels = km_final.fit_predict(X_scaled) score = silhouette_score(X_scaled, labels) print(f"Silhouette Score: {score:.3f}")

t-SNE y UMAP: Visualizando Datos de Alta Dimensión

¿Por qué necesitamos reducción dimensional?

Con 50+ variables de sensores, no podemos visualizar los datos. t-SNE y UMAP proyectan los datos en 2D/3D preservando las relaciones de proximidad entre puntos. Permite:

  • Ver si los clusters son naturalmente separables
  • Detectar outliers visualmente antes de modelar
  • Validar la calidad del clustering en 2D
  • Descubrir subestructuras no obvias en los datos

Código t-SNE

from sklearn.manifold import TSNE import matplotlib.pyplot as plt import numpy as np # Reducir a 2D para visualización tsne = TSNE(n_components=2, perplexity=30, random_state=42, n_iter=1000) X_2d = tsne.fit_transform(X_scaled) # Visualizar con colores de cluster plt.figure(figsize=(10,8)) scatter = plt.scatter(X_2d[:,0], X_2d[:,1], c=labels, cmap='tab10', s=15, alpha=0.7) plt.colorbar(scatter, label='Cluster') plt.title('Clusters en 2D (t-SNE)') plt.tight_layout()

Datos Desbalanceados: El Problema Real de la Industria

¿Por qué el desbalanceo importa?

En manufactura, las fallas son raras: 97% normal, 3% falla. Un modelo que predice siempre "normal" tiene 97% de accuracy pero 0% de utilidad. Este es el truco más común que "infla" métricas en proyectos industriales.

  • Fraude financiero: 0.1% fraudes
  • Defectos de calidad: 1-5% de piezas
  • Fallas de equipos: 2-8% del tiempo
  • Abandono de clientes: 5-20%

SMOTE: Solución con Código

from imblearn.over_sampling import SMOTE from imblearn.pipeline import Pipeline as ImbPipeline from collections import Counter print(f"Antes: {Counter(y_train)}") # {0: 9700, 1: 300} # SMOTE crea ejemplos sintéticos de la clase minoritaria # interpolando entre vecinos reales smote = SMOTE(sampling_strategy=0.5, # 1 minoritaria por 2 mayoritarias k_neighbors=5, random_state=42) X_res, y_res = smote.fit_resample(X_train, y_train) print(f"Después: {Counter(y_res)}") # {0: 9700, 1: 4850} # Mejor: dentro de un Pipeline pipe = ImbPipeline([ ('smote', SMOTE(random_state=42)), ('clf', RandomForestClassifier()) ])
💡 DATO CLAVE — SMOTE vs otras técnicas

SMOTE solo en train, nunca en test (data leakage). Alternativas: class_weight='balanced' en sklearn (más simple), ADASYN (adaptive synthetic, foco en difíciles), undersampling con RandomUnderSampler (cuando hay muchísimos datos mayoritarios). Evaluar siempre con ROC-AUC, F1 y recall de clase minoritaria.

Sesión 15 — AutoML y Metaheurísticas para Optimización Industrial

🎭 METÁFORA

AutoML es como tener un equipo de científicos de datos automático que prueba cientos de combinaciones de modelos y parámetros mientras tú duermes, y te presenta el mejor resultado al despertar. Los algoritmos genéticos son como la evolución biológica acelerada: las soluciones más aptas "sobreviven" y se "reproducen" hasta encontrar la óptima.

H2O AutoML

Plataforma AutoML de código abierto. Entrena y stacking de múltiples modelos automáticamente: GBM, XGBoost, Deep Learning, Random Forest, GLM. Escala a datos grandes. Interfaz Python sencilla.

TPOT

Tree-based Pipeline Optimization Tool. Usa programación genética para encontrar el mejor pipeline de ML (preprocesamiento + modelo). Exporta el código Python del pipeline ganador. Basado en scikit-learn.

Algoritmos Genéticos (DEAP)

Metaheurísticas inspiradas en la evolución. Excelentes para optimización combinatoria: ruteo de vehículos (VRP), programación de producción, diseño de productos, distribución de almacén.

Algoritmo Genético con DEAP: VRP Simplificado

import random from deap import base, creator, tools, algorithms # Problema: secuencia óptima de visitas a clientes CLIENTES = 20 DISTANCIAS = [[random.randint(1,50) for _ in range(CLIENTES)] for _ in range(CLIENTES)] def evaluar(individuo): """Calcula distancia total de ruta""" distancia = sum( DISTANCIAS[individuo[i]][individuo[i+1]] for i in range(len(individuo)-1)) return (distancia,) # tuple requerido por DEAP # Configurar AG creator.create("FitnessMin", base.Fitness, weights=(-1.0,)) creator.create("Individual", list, fitness=creator.FitnessMin) toolbox = base.Toolbox() toolbox.register("indices", random.sample, range(CLIENTES), CLIENTES) toolbox.register("individual", tools.initIterate, creator.Individual, toolbox.indices) toolbox.register("population", tools.initRepeat, list, toolbox.individual) toolbox.register("evaluate", evaluar) toolbox.register("mate", tools.cxOrdered) toolbox.register("mutate", tools.mutShuffleIndexes, indpb=0.1) toolbox.register("select", tools.selTournament, tournsize=3)
