Sesión 16 — Visualización de Datos: Matplotlib, Seaborn, Plotly y GeoPandas

🎭 METÁFORA

La visualización es el traductor entre datos y cerebros humanos. El cerebro procesa imágenes 60,000 veces más rápido que texto. Un gráfico bien diseñado hace en 3 segundos lo que una tabla de 500 filas no logra en 3 minutos. La pregunta no es "¿cómo dibujo esto?", sino "¿qué decisión debería generar esta visualización?"

Matplotlib — Control Total

La base de casi toda visualización Python. Máxima personalización pero verbosa. Ideal cuando necesitas control pixel a pixel: publicaciones científicas, gráficos de ingeniería con especificaciones exactas.

Seaborn — Estadística Visual

Construido sobre Matplotlib. Gráficos estadísticos de alta calidad con menos código. Integrado con DataFrames. Especialidad: distribuciones, correlaciones, comparaciones entre grupos.

Plotly — Interactividad

Gráficos interactivos (hover, zoom, filtros) en HTML. Funciona en Jupyter y web apps. Plotly Express = una línea de código. Plotly Go = control total con interactividad.

GeoPandas: Análisis Geoespacial Industrial

¿Para qué GeoPandas en Industria?

  • Logística: mapas de calor de demanda por zona geográfica
  • Retail: cobertura de tiendas, zonas sin servicio, análisis de competencia

Sesión 17 — Data Storytelling y Dashboards Power BI

🎭 METÁFORA

Los datos sin historia son como ingredientes sin receta. El data storytelling es la receta que convierte insights crudos en narrativas que mueven a la acción. La diferencia entre un dashboard que nadie usa y uno que cambia decisiones de negocio es el contexto, la narrativa y el llamado a la acción. Los datos convencen a la mente; la historia mueve a la acción.

Los 3 Elementos del Storytelling

  1. Datos: la evidencia objetiva que sustenta el mensaje
  2. Narrativa: el hilo conductor que da contexto y significado
  3. Visualización: la representación visual que hace el mensaje memorable

Estructura de una Historia de Datos

  1. Contexto: ¿cuál es la situación actual? (benchmark)
  2. Complicación: ¿qué problema o oportunidad revelan los datos?
  3. Resolución: ¿qué acción concreta se recomienda?

Principios del Diseño Visual

  • Eliminar "chartjunk" — todo lo que no agrega información
  • Un gráfico, un mensaje
  • Jerarquía visual: lo más importante, más grande y con mayor contraste
  • Color con propósito, no decorativo

Power BI: Dashboards Empresariales Profesionales

Flujo de Trabajo en Power BI

  • 1

    Power Query (ETL)

    Conectar fuentes de datos, transformar y limpiar. Interfaz visual, genera código M automáticamente.

  • 2

    Modelo de Datos

    Definir relaciones entre tablas (dimensiones y hechos). Modelo estrella para mejor performance.

  • 3

    DAX — Data Analysis Expressions

    Lenguaje de cálculo para medidas y columnas calculadas. Más potente que Excel pero similar en concepto.

  • 4

    Visualizaciones y Dashboard

    Crear reportes interactivos con filtros cruzados. Publicar en Power BI Service para acceso web.

DAX Esencial para KPIs Industriales

-- Medidas básicas Total Producción = SUM(Produccion[unidades]) Defectos Promedio = AVERAGE(Produccion[defectos]) Tasa Defectos % = DIVIDE([Total Defectos], [Total Produccion], 0) * 100 -- OEE calculado dinámicamente OEE = [Disponibilidad %] * [Rendimiento %] * [Calidad %] -- Comparación con período anterior Variación vs Mes Anterior = VAR mes_actual = [Total Producción] VAR mes_anterior = CALCULATE([Total Producción], DATEADD(Calendario[Fecha], -1, MONTH)) RETURN DIVIDE(mes_actual - mes_anterior, mes_anterior, 0) * 100 -- Top N de máquinas con más defectos Top 5 Defectos = RANKX(ALL(Maquinas), [Total Defectos], , DESC, DENSE)

Sesión 18 — UX/UI y Despliegue de Apps Web con Streamlit

🎭 METÁFORA

Streamlit es como tener un equipo de front-end incluido en tu librería de Python. Escribes lógica de datos en Python puro y Streamlit se encarga de convertirlo en una aplicación web interactiva. Sin HTML, sin JavaScript, sin CSS. Un científico de datos puede desplegar una app funcional en 30 minutos que antes requería semanas de desarrollo.

Principios UX para Apps de Datos

Affordance: que sea obvio cómo se usa. Feedback: el usuario siempre sabe qué está pasando (spinners, mensajes). Error prevention: validar inputs antes de procesar. Consistency: mismo lenguaje visual en toda la app.

Cuándo usar Streamlit

Prototipado rápido de herramientas analíticas internas. Demo de modelos ML para stakeholders. Dashboards dinámicos con predicciones en tiempo real. Herramientas de exploración de datos para equipos no técnicos.

Cuándo usar otras herramientas

Power BI/Tableau: dashboards BI para directivos con actualizaciones automáticas. FastAPI + React: apps de producción con alta carga y personalización. Dash (Plotly): más control que Streamlit, layouts complejos.

Sesión 19 — PySpark y Big Data: Las 5 Vs de los Datos Masivos

🎭 METÁFORA

Pandas es como una mesa de trabajo en tu oficina: rápida, conveniente, pero con capacidad limitada. PySpark es como una fábrica con 1,000 trabajadores en paralelo: cada uno procesa una parte del problema simultáneamente. Cuando tus datos no caben en la RAM de tu computadora (>16GB), necesitas la fábrica.

Volumen

Datos que no caben en RAM. Terabytes o petabytes. Historial de transacciones, logs de IoT (millones de lecturas/día), imágenes médicas de hospitales, secuencias genómicas.

Velocidad

Datos que se generan o procesan en tiempo real. Telemetría de sensores cada milisegundo, transacciones financieras, feeds de redes sociales. Spark Streaming procesa en micro-batches.

Variedad + Veracidad + Valor

Variedad: structured, semi-structured, unstructured. Veracidad: calidad y confiabilidad de los datos (sensores que fallan, duplicados). Valor: el resultado final: decisiones e inteligencia de negocio.

Sesión 20 — Machine Learning a Escala con PySpark MLlib

🎭 METÁFORA

MLlib es como tener el poder de scikit-learn pero distribuido en una flota de computadoras. Cuando tienes 50 millones de registros de clientes y quieres segmentarlos con K-Means, un solo computador tardaría días; MLlib lo hace en minutos repartiendo el trabajo en paralelo. La misma idea, escala infinita.

MLlib Pipeline

Igual que sklearn: Pipeline de etapas (Transformer + Estimator). StringIndexer → OneHotEncoder → VectorAssembler → modelo. Fit en train, transform en test. Portabilidad garantizada.

Algoritmos Incluidos

Regresión lineal/logística, Random Forest, Gradient Boosted Trees, K-Means, Bisecting K-Means, PCA, TF-IDF + Word2Vec (NLP), Naive Bayes, SVM lineal, ALS (recomendación).

Cuándo usar MLlib vs scikit-learn

MLlib: datos > 10GB, necesitas procesamiento distribuido, cluster disponible. scikit-learn: datos en RAM, prototipado rápido, más algoritmos avanzados. Para producción masiva: MLlib.

Pipeline MLlib: Clasificación Distribuida

Clasificación con Random Forest en Spark

from pyspark.ml import Pipeline from pyspark.ml.feature import (VectorAssembler, StringIndexer, StandardScaler) from pyspark.ml.classification import RandomForestClassifier from pyspark.ml.evaluation import BinaryClassificationEvaluator # 1. Preparar features indexer = StringIndexer(inputCol="turno", outputCol="turno_idx") assembler = VectorAssembler( inputCols=["temperatura","vibracion", "presion","turno_idx"], outputCol="features_raw") scaler = StandardScaler(inputCol="features_raw", outputCol="features") # 2. Modelo rf = RandomForestClassifier( featuresCol="features", labelCol="falla", numTrees=200, maxDepth=10) # 3. Pipeline completo pipeline = Pipeline(stages=[indexer, assembler, scaler, rf]) # 4. Train/test split train, test = df.randomSplit([0.8, 0.2], seed=42) model = pipeline.fit(train) preds = model.transform(test) # 5. Evaluación evaluator = BinaryClassificationEvaluator( labelCol="falla", metricName="areaUnderROC") auc = evaluator.evaluate(preds) print(f"AUC: {auc:.4f}")
